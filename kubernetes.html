Overview
Why Kubernetes?
Docker runs containers, but Kubernetes manages them at scale. It provides automatic self-healing, load balancing, and rolling updates.
🔄
Self-Healing
K8s restarts failed containers, replaces dead pods, and reschedules when nodes go down — automatically.
⚖️
Load Balancing
Traffic is distributed across all 3 running pods. No single pod gets overloaded.
📈
Scaling
Change replicas from 3 to 10 with one command. K8s creates new pods instantly.
❤️
Health Probes
livenessProbe and readinessProbe check /health every 10s. Failed pods are restarted immediately.

Code
deployment.yaml
Tells Kubernetes to always keep 3 replicas of the Flask app running, with health probes.
deployment.yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: healing-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: healing-app
  template:
    metadata:
      labels:
        app: healing-app
    spec:
      containers:
      - name: healing-app
        image: healing-app:latest
        ports:
        - containerPort: 5000
        livenessProbe:
          httpGet:
            path: /health
            port: 5000
          initialDelaySeconds: 10
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 5000
          initialDelaySeconds: 5
          periodSeconds: 5
Code
service.yaml
Exposes the deployment as a network service so it can be accessed from outside the cluster.
service.yaml 
apiVersion: v1
kind: Service
metadata:
  name: healing-app-service
spec:
  selector:
    app: healing-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 5000
  type: NodePort
Commands
Essential kubectl Commands
Terminal 
# Apply deployment and service
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

# Check pod status
kubectl get pods

# Watch pods in real-time
kubectl get pods -w

# Simulate crash — delete a pod (K8s recreates it!)
kubectl delete pod healing-app-688885fcc9-bz8w5

# View pod logs
kubectl logs healing-app-688885fcc9-bz8w5

# Describe pod (shows events + health probes)
kubectl describe pod healing-app-688885fcc9-bz8w5

# Scale replicas up/down
kubectl scale deployment healing-app --replicas=5
