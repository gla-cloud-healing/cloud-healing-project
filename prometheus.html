Prometheus — Metrics
Monitoring

Prometheus is an open-source monitoring system that scrapes metrics from our Flask app every 15 seconds and stores them in a time-series database for analysis and alerting.

Overview
What does Prometheus do?
🔄
Pull-based Scraping
Prometheus pulls metrics from our /metrics endpoint every 15s instead of apps pushing data to it.
📊
Time-Series DB
All metrics stored with timestamps — allowing historical analysis and trend detection.
🔍
PromQL
Powerful query language to analyze metrics. rate(), sum(), avg() — used in Grafana dashboards.
🔔
Alerting
Define alert rules that fire when metrics cross thresholds — integrates with Alertmanager.

Code
Flask app — /metrics endpoint (app.py)
This is how our Flask app exposes metrics for Prometheus to scrape.
app/app.py — metrics setup 
from flask import Flask
from prometheus_client import Counter, Histogram, generate_latest

app = Flask(__name__)

# Counter — total requests (only goes up)
REQUEST_COUNT = Counter(
    'app_request_count_total',
    'Total HTTP Requests',
    ['method', 'endpoint', 'status']
)

# Histogram — request duration in seconds
REQUEST_LATENCY = Histogram(
    'app_request_latency_seconds',
    'Request latency in seconds',
    ['endpoint']
)

# Prometheus scrapes this endpoint every 15s
@app.route('/metrics')
def metrics():
    return generate_latest(), 200
PromQL Queries
Useful queries for our project
PromQL — use at localhost:9090 
# Total requests since startup
app_request_count_total

# Request rate per minute (last 1 min)
rate(app_request_count_total[1m])

# Average request latency
rate(app_request_latency_seconds_sum[1m])
  / rate(app_request_latency_seconds_count[1m])

# 95th percentile latency
histogram_quantile(0.95,
  rate(app_request_latency_seconds_bucket[5m]))
