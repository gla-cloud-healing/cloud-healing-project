Overview
What is Docker?
Docker is a platform that uses OS-level virtualization to deliver software in isolated packages called containers. Unlike VMs, containers share the host OS kernel — making them fast and lightweight.
📦
Isolation
Each container has its own filesystem, processes, and network. No conflicts between apps running on the same machine.
⚡
Lightweight
Containers start in milliseconds and use far less RAM than VMs. Our healing-app container runs in ~200MB RAM.
🔁
Reproducibility
Same Dockerfile = same environment everywhere. No "works on my machine" problems across team members.
🔧
Self-Healing
Docker's restart policies and our healing_script.py restart containers automatically when they crash.
Architecture
How Docker fits in our system
Docker is the base layer — everything else (K8s, Prometheus, Grafana) runs inside Docker containers.

Code
Dockerfile — app/Dockerfile
Defines how to build the Flask app container step by step from a base Python image.
app/Dockerfile 
# Base image — lightweight Python 3.11
FROM python:3.11-slim

# Set working directory inside container
WORKDIR /app

# Copy dependency list first (layer caching)
COPY requirements.txt .

# Install Python dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Copy application source code
COPY . .

# Expose Flask port
EXPOSE 5000

# Start the Flask application
CMD ["python", "app.py"]
Code
docker-compose.yml — Full Stack
Starts all 3 services (Flask app, Prometheus, Grafana) with one command on a shared network.
docker-compose.yml 
version: '3.8'

services:
  my-app:
    build: ./app
    container_name: my-app
    ports:
      - "5000:5000"
    restart: unless-stopped
    networks:
      - monitoring

  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml
    networks:
      - monitoring

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
    networks:
      - monitoring

networks:
  monitoring:
    driver: bridge
Commands
Essential Docker Commands
Terminal 
# Build image from Dockerfile
docker build -t healing-app ./app

# Start all services (detached mode)
docker-compose up -d

# Stop all services
docker-compose down

# View running containers
docker ps

# View container logs
docker logs my-app

# Manually restart the app container
docker restart my-app

# Execute shell inside container
docker exec -it my-app bash

# Check container resource usage
docker stats
