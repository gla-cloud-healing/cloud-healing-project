Docker Compose — Service
Integration

Docker Compose is the glue that holds everything together. One command starts all 3 services (Flask app, Prometheus, Grafana) on a shared network — ready to communicate by service name.

docker-compose.yml docker-compose up -d monitoring network
Quick Reference

View Logsdocker-compose logs
Servicesmy-app, prometheus, grafana
Networkmonitoring (bridge)
Overview
Why Docker Compose?
⚡
One Command
docker-compose up -d starts all 3 services at once. No need to run 3 separate docker run commands.
🌐
Shared Network
All containers join the "monitoring" network and talk to each other using service names instead of IPs.
💾
Volume Mounts
Config files (prometheus.yml) mounted into containers without rebuilding images.
🔁
Restart Policy
unless-stopped ensures containers auto-restart after EC2 reboot or system restart.

All 3 services are on the same monitoring bridge network — they use service names to talk to each other.

Code
docker-compose.yml — Full File
docker-compose.yml 
version: '3.8'

services:

  my-app:
    build: ./app
    container_name: my-app
    ports:
      - "5000:5000"
    restart: unless-stopped
    networks:
      - monitoring

  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml
    networks:
      - monitoring

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin123
    volumes:
      - grafana-storage:/var/lib/grafana
    networks:
      - monitoring

volumes:
  grafana-storage:

networks:
  monitoring:
    driver: bridge
Commands
Essential Compose Commands
Terminal 
# Start all services in background
docker-compose up -d

# Stop and remove containers
docker-compose down

# View logs for all services
docker-compose logs

# View logs for specific service
docker-compose logs my-app

# Check service status
docker-compose ps

# Rebuild after code changes
docker-compose up -d --build
